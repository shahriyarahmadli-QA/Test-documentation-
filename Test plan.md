## 1. Məqsəd
Bu test planı layihənin funksionallığını, performansını və istifadəçi təcrübəsini yoxlamaq üçün hazırlanmışdır. Məqsəd — səhvləri erkən aşkar etmək və məhsulun keyfiyyətini təmin etməkdir.

## 2. Testin Əhatə Dairəsi
- **Frontend**: UI elementləri, naviqasiya, formaların işləməsi
- **Backend**: API cavabları, verilənlər bazası əməliyyatları
- **Security**: İstifadəçi giriş/çıxış, məlumatların qorunması
- **Performance**: Yüklənmə sürəti, sistemin dayanıqlılığı

## 3. Test Strategiyası
- **Manual Testing**: Əsas funksiyaların əl ilə yoxlanması
- **Automated Testing**: Unit və integration testlər
- **Regression Testing**: Yeni dəyişikliklərdən sonra əvvəlki funksiyaların işləməsini yoxlamaq
- **Acceptance Testing**: İstifadəçi tələblərinə uyğunluğun təsdiqi

## 4. Test Mühiti
- **OS**: Windows 11, Ubuntu 22.04
- **Browser**: Chrome, Firefox, Edge
- **Database**: PostgreSQL 15
- **Tools**: GitHub Actions, Jest, Selenium

- ## 5. Test Tapşırıqları
- Login funksiyasını yoxlamaq
- Yeni istifadəçi qeydiyyatını test etmək
- API cavablarının düzgünlüyünü yoxlamaq
- UI elementlərinin mobil cihazlarda uyğunluğunu test etmək

## 6. Risklər
- İnfrastrukturun qeyri-sabitliyi
- Test mühitinin istehsal mühitindən fərqlənməsi
- Resurs çatışmazlığı (vaxt, insan gücü)

## 7. Çatdırılmalar
- Test Case sənədləri
- Test nəticələrinin hesabatı
- Bug report-lar
- Final test nəticələri

## 8. Cədvəl
- **Həftə 1-2**: Əl testləri
- **Həftə 3**: Avtomatlaşdırılmış testlərin yazılması
- **Həftə 4**: Regression və acceptance testləri
